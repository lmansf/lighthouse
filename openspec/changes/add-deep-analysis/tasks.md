## 2. The `investigate` engine + in-vault write (engine, then write)
- [x] 2.1 `investigate(table, files, is_cloud) -> Report`: build the catalog + register the tables (the `insights::scan` pattern — `catalog::columns_for` + `analytics::register_tables` over the included tabular files), resolve the target `table`'s registered SQL name + typed columns. (Takes the resolved `(id, name, path)` triples like `insights::scan`, keeping `reports.rs` free of `vault::` id-resolution — the op gathers them, §4.1.) A Date+Numeric shape gate up front: a non-analyzable table returns an honest empty report BEFORE the always-applicable data-quality audit could turn it into a spurious one-section report.
- [x] 2.2 For each `recipes::BUILTINS` whose `.applicable(cols)` holds, resolve params + run the representative query (`plan[0]`) via `analytics::run_query`; collect `(recipe.name, recipe.summary, QueryResult, sql)` as a sub-analysis with a `section_headline` (engine cells → one line, the `insights` discipline). A recipe that doesn't resolve or returns no rows is SKIPPED (`run_query` returns `Err` on zero rows, so the `let Ok(res)` skip is free). Bounded: the fixed ≤7 builtins, one representative query each.
- [x] 2.3 Caveats: an always-true month-bucketing structural caveat + the last section's STRUCTURAL assumptions (`ledger::assumption_ledger_parts` with `rows: None`, so the per-query row count — misleading for a `LIMIT 1` argmax — is omitted) + a data-quality finding (the worst-null column); then `assemble`. Degradation: no Date+Numeric shape (or no sections) → an honest empty report, never an error. (Also fixed the data-quality-audit recipe's `UNION ALL` to `ORDER BY column_name` so its result row-order is stable — the report render must be byte-identical.)
- [x] 2.4 In-vault write (the `exportChat`/briefing precedent): `write_report(report, investigation_id)` renders → a write-artifact allowlist subdir (`investigations::notes_subdir(id)` when an investigation is named, else the `Lighthouse Reports` allowlist entry) + `vault::write_artifact`; returns `(savedId, savedName)`. Non-egress, sanitized, never-overwrite.
- [ ] 2.5 OPTIONAL model narrative (opt-in, never the core): DEFERRED by the model-free design decision — the deterministic report stands complete without it and the CI floor runs model-free only. The op exposes no opt-in yet; a bounded `stream_answer` prose intro is a clean follow-on that changes no wire shape.
- [x] 2.6 Tests (`tests/reports_test.rs`): `investigate` over a fixture Date+Numeric table returns a multi-section report whose sections are the applicable recipes' verified results and whose every summary figure appears in a section result; an unanalyzable (and an unknown) table returns an empty-sections honest report; the render is byte-identical across two runs with a fixed time; the write lands under the `Lighthouse Reports` allowlist subdir and returns an id. VERIFIED (5 pass).
